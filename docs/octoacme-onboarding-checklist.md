## Purpose
Provide a standardized checklist to ensure new team members are properly onboarded and can contribute effectively to OctoAcme projects.

## Who uses this
- Project Managers (to coordinate onboarding)
- Team Leads (to facilitate technical setup)
- New team members (to track their own progress)

---

## Pre-Day 1 (Before Start Date)

### Administrative Setup
- [ ] Workspace and equipment ordered
- [ ] System accounts provisioned (email, Slack, GitHub, etc.)
- [ ] Access to project repositories granted
- [ ] Calendar invites sent for recurring team meetings

### Documentation Prepared
- [ ] Welcome email sent with first-day logistics
- [ ] Team roster and contact information shared
- [ ] Link to project documentation repository provided

---

## Week 1: Foundation & Context

### Orientation
- [ ] Welcome meeting with Project Manager
- [ ] Introduction to team members and their roles
- [ ] Overview of current projects and priorities
- [ ] Review organizational structure and escalation paths

### Documentation Review
- [ ] Read Project Management Overview (`octoacme-project-management-overview.md`)
- [ ] Review Roles and Personas document (`octoacme-roles-and-personas.md`)
- [ ] Understand project lifecycle phases
- [ ] Review team communication cadences and tools

### Technical Setup
- [ ] Development environment set up and verified
- [ ] Access to all required tools and systems confirmed
- [ ] Code repositories cloned and builds successful
- [ ] Familiarize with branching and PR conventions

### First Assignments
- [ ] Shadow team members in daily standups
- [ ] Attend sprint planning or backlog refinement
- [ ] Pick up first small task (e.g., documentation update, minor bug fix)

---

## Week 2: Deeper Integration

### Process Immersion
- [ ] Review Risk Management & Communication processes (`octoacme-risks-and-communication.md`)
- [ ] Understand Definition of Done and acceptance criteria standards
- [ ] Learn about testing and quality standards
- [ ] Review change control processes

### Active Participation
- [ ] Complete and submit first pull request
- [ ] Participate in code review (as reviewer or reviewee)
- [ ] Attend sprint demo or milestone review
- [ ] Contribute to daily standup discussions

### Relationship Building
- [ ] Schedule 1:1 meetings with key collaborators
- [ ] Join team social activities or informal chats
- [ ] Ask questions and seek clarifications proactively

---

## Week 3-4: Autonomy & Contribution

### Ownership
- [ ] Take ownership of user story or feature backlog item
- [ ] Participate in estimation and planning discussions
- [ ] Contribute to technical design or architecture discussions
- [ ] Update project documentation as needed

### Quality & Collaboration
- [ ] Write tests for new code contributions
- [ ] Provide code reviews for teammates
- [ ] Help identify or mitigate risks in current sprint
- [ ] Share learnings or insights with the team

### Feedback Loop
- [ ] 30-day check-in with Project Manager
- [ ] Provide feedback on onboarding experience
- [ ] Identify any gaps in documentation or processes
- [ ] Discuss growth goals and learning opportunities

---

## Role-Specific Onboarding Extensions

### For Developers
- [ ] Complete codebase walkthrough with senior developer
- [ ] Review architectural decision records (ADRs)
- [ ] Set up and run full test suite locally
- [ ] Deploy to local/dev environment successfully

### For Risk Managers
- [ ] Review current Risk Register and risk history
- [ ] Shadow existing risk review meetings
- [ ] Understand escalation procedures for critical risks
- [ ] Set up risk monitoring tools and dashboards

### For Stakeholder Liaisons
- [ ] Meet with key stakeholders (introductions)
- [ ] Review stakeholder communication plan
- [ ] Understand stakeholder feedback mechanisms
- [ ] Shadow stakeholder update meetings

### For Change Control Coordinators
- [ ] Review change control policy and procedures
- [ ] Access change management system/tools
- [ ] Review historical change requests and decisions
- [ ] Shadow Change Control Board meeting

### For Product Managers
- [ ] Review product vision and roadmap
- [ ] Understand customer personas and use cases
- [ ] Access analytics and metrics dashboards
- [ ] Meet with customer-facing teams (sales, support)

### For Project Managers
- [ ] Review active project charters and plans
- [ ] Access project tracking tools and dashboards
- [ ] Understand resource allocation and capacity planning
- [ ] Review historical retrospectives and lessons learned

---

## 90-Day Milestone

### Expected Outcomes
- [ ] Fully autonomous contributor to team deliverables
- [ ] Comfortable with all project processes and tools
- [ ] Established working relationships with team and stakeholders
- [ ] Contributing to team improvements and retrospectives

### Review Meeting
- [ ] Conduct 90-day review with Project Manager
- [ ] Discuss accomplishments and areas for growth
- [ ] Set goals for next quarter
- [ ] Update team documentation based on onboarding insights

---

## Continuous Improvement

### Feedback for Process
- What worked well in your onboarding?
- What could be improved?
- What documentation was missing or unclear?
- What would you recommend for future new hires?

### Action Items
- [ ] Document improvements to onboarding process
- [ ] Update checklists based on role-specific needs
